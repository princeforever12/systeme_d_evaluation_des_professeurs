# Plan de gestion de projet (V1)
## Projet tutoré L2 — UTT-LOKO ITER

## 1. Équipe
- 5 étudiants (groupe projet)
- 1 encadrant pédagogique

## 2. Organisation (sprints)
### Sprint 1 — Cadrage (Semaine 1)
- Finalisation cahier des charges
- Validation questionnaire
- Priorisation backlog

### Sprint 2 — Implémentation cœur (Semaines 2-3)
- Flux étudiant filière/classe/matière
- Questionnaire 4 dimensions
- Stockage DB + migration

### Sprint 3 — Reporting et recette (Semaine 4)
- Rapport filière/classe/matière
- Tests fonctionnels
- Corrections

### Sprint 4 — Soutenance (Semaine 5)
- Démo
- Slides
- Bilan et perspectives (anonymat par token, dashboard avancé)

## 3. Gestion des risques
- Risque: retard de validation du questionnaire
  - Mitigation: réunion de validation à J+2
- Risque: régression DB lors des migrations
  - Mitigation: script de migration non destructif + sauvegarde
- Risque: faible couverture tests
  - Mitigation: checklist de recette et tests de routes clés

## 4. Livrables
- Code source applicatif
- Cahier des charges V1
- Plan de gestion de projet V1
- Rapport de tests
- Support de soutenance
